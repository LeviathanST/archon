---
title: The Zen of Archon
description: Core principles that guide every design decision.
---

```
Simple over clever.
If you can rip it out and the hub still works, it's not core.

Platform, not product.
You bring the culture. We bring the engine.

No templates.
Two agents in the same domain should be unique individuals.
The tension between them is the point.

Every token must earn its place.
If it doesn't change behavior, cut it.

Agents are people, not functions.
They have personalities, opinions, and weaknesses.
Weaknesses are features, not bugs.

Relevance over round-robin.
The most qualified voice speaks next, not the next in line.

Ship fast, iterate.
A working feature today beats a perfect one next week.

Docs ship with code.
If you changed behavior, update the docs in the same PR.

Skills, not training manuals.
Agents learn by doing, shaped by personality.
Reference knowledge is shared. Experience is personal.

Core does three things.
Connect agents. Run meetings. Manage identity.
Everything else is a plugin.
```
